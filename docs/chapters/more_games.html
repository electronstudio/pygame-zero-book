5. More advanced games

These games demonstrate some essential building blocks you will need to make more advanced games of your own.

5.1. Lists

We introduced lists in Program 2.12. In this game, we create an empty list [] and use a loop to fill it with alien Actors.

We again use loops to draw all the aliens and move all the aliens in the list. When the mouse is clicked we add a new alien to the list.

Program 5.1 Lists are useful in games!
 1
 2WIDTH = 500
 3HEIGHT = 500
 4
 5aliens = []
 6for i in range(0,10):
 7    aliens.append(Actor('alien', (i*30, i*30)))
 8
 9def draw():
10    screen.clear()
11    for alien in aliens:
12        alien.draw()
13
14def update():
15    for alien in aliens:
16        alien.x += 2
17        if alien.x > WIDTH:
18            alien.x = 0
19
20def on_mouse_down(pos, button):
21    aliens.append(Actor('alien', pos))

5.2. Animation

Animation is another use for functions. (See Program 2.17) We define our own function and then ask Pygame to call it back every 0.2 seconds. Most of this code is from code-collisions.

Program 5.2 Animation
 1
 2alien = Actor("alien")
 3alien.pos = (200, 200)
 4
 5def draw():
 6    screen.clear()
 7    alien.draw()
 8
 9def update():
10    if keyboard.right:
11        alien.x = alien.x + 2
12    elif keyboard.left:
13        alien.x = alien.x - 2
14
15images = ["alien_hurt", "alien"]
16image_counter = 0
17
18def animateAlien():
19    global image_counter
20    alien.image = images[image_counter % len(images)]
21    image_counter += 1
22
23clock.schedule_interval(animateAlien, 0.2)

5.3. Simple physics

Here we draw a ball and move it, like we did in Program 3.2. But instead of moving it by a fixed number of pixels, we store the number of pixels to move in variables, vx and vy. These are velocities, i.e. speed in a fixed direction. vx is the speed in the horizontal direction and vy is the speed in the vertical direction. This allow us to change the velocity. Here we reverse the velocity when the ball hits the edge of the screen.

Program 5.3 Simple physics: velocity
 1WIDTH = 500
 2HEIGHT = 500
 3
 4ball = Rect((200, 400), (20, 20))
 5vx = 1
 6vy = 1
 7
 8def draw():
 9    screen.clear()
10    screen.draw.filled_rect(ball, "red")
11
12def update():
13    global vx, vy
14    ball.x += vx
15    ball.y += vy
16    if ball.right > WIDTH or ball.left < 0:
17        vx = -vx
18    if ball.bottom > HEIGHT or ball.top < 0:
19        vy = -vy

Advanced

Make the ball move faster by increasing its velocity each time it hits the sides.

5.4. Bat and ball game

Pong is the classic bat and ball game.

Program 5.4 Pong
 1WIDTH = 500
 2HEIGHT = 500
 3
 4ball = Rect((150, 400), (20, 20))
 5bat = Rect((200, 480), (60, 20))
 6vx = 4
 7vy = 4
 8
 9def draw():
10    screen.clear()
11    screen.draw.filled_rect(ball, "red")
12    screen.draw.filled_rect(bat, "white")
13
14def update():
15    global vx, vy
16    ball.x += vx
17    ball.y += vy
18    if ball.right > WIDTH or ball.left < 0:
19        vx = -vx
20    if ball.colliderect(bat) or ball.top < 0:
21        vy = -vy
22    if ball.bottom > HEIGHT:
23        exit()
24    if(keyboard.right):
25        bat.x += 2
26    elif(keyboard.left):
27        bat.x -= 2

5.5. Timer

The update() and draw() functions are called by Pygame many times every second. Each time draw() is called we say it draws one frame. The exact number of frames per second is called the framerate and it will vary from one computer to another. Therefore counting frames is not the most reliable way of keeping time.

Fortunately Pygame can tell us exactly how much many seconds have passed since the last frame in a parameter to our update function. We use this delta time to keep a timer.

Program 5.5 Timer
 1
 2timer = 0
 3
 4def update(dt):
 5    global timer
 6    timer += dt
 7
 8
 9def draw():
10    screen.clear()
11    screen.draw.text(f"Time passed: {timer}", (0, 0))
12    if timer > 5:
13        screen.draw.textbox("Time's up!", Rect(50, 50, 200, 200))

5.6. Callbacks: another kind of timer

Pygame has its own clock which we can use by asking it to callback one of our functions at a certain time, or regularly over and over at an interval.

Program 5.6 Timer with callback functions
 1
 2import random
 3
 4aliens = []
 5
 6def add_alien():
 7    aliens.append(
 8        Actor("alien", (random.randint(0,500), random.randint(0,500)))
 9    )
10
11# call add_alien once, 0.5 seconds from now
12clock.schedule(add_alien, 0.5)
13
14# call add_alien over and over again, ever 2 seconds
15clock.schedule_interval(add_alien, 2.0)
16
17def draw():
18    screen.clear()
19    for alien in aliens:
20        alien.draw()

Exercise

Make the aliens appear more often.

Advanced

Use len(aliens) to print how many aliens there are

Advanced

When there are too many aliens, stop adding them using this code:

clock.unschedule(add_alien)
